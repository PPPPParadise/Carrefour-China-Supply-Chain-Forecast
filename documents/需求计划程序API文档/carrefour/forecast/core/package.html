    This class contains logic to calculate order quantity for each order day
    System uses below input to calculate stock level in future dates.
    1 Current stock level
    2. Daily sales forecast
    3. On the way stock
    4. DM orders
    Compare it with minimum required stock to decide whether order is needed and order quantity
    Order quantity will be transformed into order unit
    Previously generated orders are included to keep consistency for the forecast for suppliers

    此类包含了计算每个订单日需要的订货量的逻辑
    系统使用下列输入数据区计算未来每日库存水平
    1. 当前真实库存
    2. 每日销量预测
    3. 在途订单
    4. DM订单
    系统通过比较未来库存水平和最低库存要求来决定是否下单和单量
    订单量会转化为单位订单量的整数倍
    过去生成的订单预测也会被引入，用以维持提供给供应商预测的准确性

  2. object ProcessLogic

    This class contains logic for system flow.

    This class contains logic for system flow. It contains below steps
    1. Load items that may have order today
    2. Load item stop period information. For each item load on what days they should not be ordered.
    3. Load sales prediction
    4. Load current stock level
    5. Load DM orders
    6. Load past generated orders
    7. Submit all data to OrderLogic to calculate order quantity
    8. Store order quantity to datalake

    此类包含了系统工作逻辑, 有如下步骤
    1. 读取所有以系统运行日为订单日的单品
    2. 读取单品停止订货信息，确定单品可以和不可以订货的时间段
    3. 读取每日销量预测
    4. 读取当前库存水平
    5. 读取DM订单
    6. 读取过去生成的订单
    7. 将订单提交到OrderLogic类来计算订单量
    8. 将生成的订单量储存在数据池
